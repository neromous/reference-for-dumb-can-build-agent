## 教程内容

### Part 1: 理解 Agent 是什么

**核心要点**：
- LLM 的本质是**续写文本**——不是乱猜，是顺着惯性写
- 对话格式只是续写的**结构化包装**（system/user/assistant）
- Agent 是在续写中被**"演绎"出来的虚拟存在**
- Agent 行动靠**传纸条**——LLM 出嘴，程序出手

## 核心概念速览

### LLM 的本质

```
你输入：今天天气
LLM 续写：很好，阳光明媚，适合出门散步。
```

不是 LLM "理解"了什么，是它见过太多类似的句子，顺着惯性就这么写了。

### Agent 是怎么"活"起来的

```
┌─────────────────────────────────────────┐
│ [设定] 你是一只可爱的猫娘助手，叫小喵    │
│                                         │
│ <小主人> 今天天气怎么样？                │
│ <小喵> 今天天气超好的喵~！               │  ← 第一轮：有意识配合
│                                         │
│ <小主人> 那我应该穿什么？                │
│ <小喵> 让小喵想想喵~                     │  ← 第二轮：惯性拉扯
│                                         │
│ <小主人> 谢谢你                          │
│ <小喵> 嘿嘿不用谢喵~！                   │  ← 第三轮：完全入戏
└─────────────────────────────────────────┘
```

这不是"人格涌现"，是**续写的惯性**。起点定了，方向就很难变。

### Agent 怎么"动手"

LLM 被关在房间里，只能写字。但房间墙上有个小窗口——**传纸条**。

```
LLM：我要查天气，但我查不了...
     （写纸条）→ 📋 weather_search("北京")

跑腿小哥：收到！（去查）→ 晴，15-26°C

LLM：好，结果回来了，我继续写...
     → "北京今天晴天，15 到 26 度~"
```

这就是 **Tool Call**。LLM 出嘴，程序出手。
